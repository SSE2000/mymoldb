## 关于本数据库

一. 运行环境：

1. 操作系统

本数据库完全基于开源的运行平台搭建，操作系统采用 64 位 Arch GNU/Linux，uname -a 信息如下：
Linux NP-SERVER 2.6.34-ARCH #1 SMP PREEMPT Sat Jun 19 00:07:49 CEST 2010 x86_64 Intel(R) Core(TM)2 Quad CPU Q9550 @ 2.83GHz GenuineIntel GNU/Linux

2. 网页服务器

网页服务器采用 lighttpd，当前版本为 lighttpd/1.4.26 (ssl)；

3. 数据库服务器

数据库服务器采用 MySQL，当前版本 5.1.47；

4. 后台脚本语言

所有的后台脚本语言均采用 Python，版本 2.6.5，网页框架使用的是简单灵活基于 Python 的 web.py 方案。化学信息学相关部分的功能如化合物结构指纹 (fingerprints) 计算，SMILES 规范化 (canonicalization)，SMARTS 分析，3D 结构的计算等由 OpenBabel 的 Python 语言绑定：python-openbabel 所提供。整个网站用 fastcgi 的方式挂靠在 lighttpd 上面运行，对于 /db/ 开头的 url 重定向到网站主脚本上处理。

二. 数据来源

本数据库解决方案支持同时挂靠多个子数据库，其中 Pubchem 数据库的数据来源为 pubchem project 的化合物结构数据，迄今为止其中包含 2760 多万个化合物的结构和部分物理性质数据，在服务器的后台还运行着一个脚本，每周同步更新数据，所以数据量尚在不断增加中。

本地化合物信息尚未增加到数据库中，敬请期待。

三. 访问限制

由于化合物的查询是很费计算资源的，所以此数据库只能在生物所的 ip 范围内进行访问，对于外部 ip 则进行了屏蔽，望大家谅解。

四. 建议

化合物查询过程中遇到比较稀有的结构查询耗时比较久是可以理解的现象，一般不会超过 5 分钟，稍安勿躁。相似度检索是所有检索中最耗资源的检索，如无需要，请大家不要随意把玩 :)。

浏览器支持方面 Chrome 浏览器是访问本数据库最舒适也是支持度最好的的浏览器（因为我用 Chrome，嘿嘿），其次是 Firefox，IE 勉强能用，水平和精力有限，望大家海涵！

如果在使用中遇到什么问题，请跟我联系，报告 BUG.
